## Question

截至 `2026-04-04` 当前 worktree，`TEST-COST-001` 的“缺 dedicated canonical suite / strict mapping”这一类 blocker 已不再成立：

* `tests/pre-release/test-cost-001.test.mjs` 已存在；
* `packages/testing/src/evidence.mjs` 已把 `TEST-COST-001 -> pre-release` canonical mapping 接上 strict parser；
* `packages/testing/src/nonlocal.mjs` 已把 `pre_release_cost_observation` sidecar parsing 接到 fail-closed environment gate；
* 对应本地回归已验证“malformed sidecar / missing sidecar / unexpected sidecar”都会继续 fail-closed。

但这并不等于 `08.04D` 或 `EVID-COST-001` 可以诚实转为完成。复核当前官方 Cloudflare 文档后，当前 repo 仍缺一条能满足 [41-observability-performance-and-cost.md](/root/Matrix/spec/framework/41-observability-performance-and-cost.md) `6.4` 的官方、billing-safe、environment-scoped pre-release proof path：

1. Cloudflare 官方 GraphQL Analytics 文档把该 surface 定位为 aggregated analytics，并明确说明不要拿它做 billing。
2. Workers 官方 metrics surface 只暴露 request count 与 CPU/latency percentiles，而不是 bounded workload window 下的 billable total CPU；因此不能把 `p50 * request_count` 之类估算值写成 `actual_total_usd`。
3. Durable Objects 官方 pricing 按 wall-clock duration `GB-s` 计费，但官方 Durable Objects analytics surface 暴露的是 `requests`、`cpuTime` 与 `storedBytes`；当前没有被证明可把 `cpuTime` 诚实换算成 billed duration。
4. Cloudflare Billing Usage / PayGo API 仍有 beta / select-account 适用边界，而且当前已知 surface 是 account-scoped，而不是经证明可对当前 pre-release environment / bounded workload window 直接出具 truth snapshot。

这条 blocker 已在 latest reviewed pushed-head run `23984768921`（head `326e5d886ce615424d4fac8fc5e592cc985e84ef`）上再次获得 same-head remote 复核：dedicated canonical `tests/pre-release/test-cost-001.test.mjs` 已真实执行，raw `pre-release.log` 仍记录 `/_ops/v1/cost/observation` 返回 `500/internal`，消息明确指出 `TEST-COST-001 pre-release proof remains fail-closed ... see OQ-0006`；同轮 `pre_release_cost_observation` sidecar 继续缺失，而 pre-release attestation 也继续因为同环境内的 `TEST-OPS-001` 失败而被跳过。这说明当前 remaining gap 仍然不是 suite / mapping wiring，而是官方 surface 本身仍不足以诚实产出 `PreReleaseCostObservation`。

因此，当前 repo 先前那条“从 GraphQL analytics + pricing table 直接拼出 `actual_total_usd`，再把 `modeled_total_usd` 设成同一个值”的 success path 不能保留。当前 worktree 已把 `/_ops/v1/cost/observation` 改回诚实的 fail-closed blocker：它会先验证 pre-release 环境、必需配置与 Workers `usage_model == standard` 前提，然后明确拒绝伪造 `PreReleaseCostObservation`。

换言之，当前 remaining gap 已从“没有 contract / 没有 suite / 没有 mapping”收敛为更精确的 truth：

* dedicated canonical suite 与 strict mapping 已存在；
* parser / harness gating 已存在且继续 fail-closed；
* 但官方 Cloudflare surfaces 仍不足以让当前 repo 诚实产出 `actual_total_usd` + `modeled_total_usd` + `drift_ratio` 均可信的 `PreReleaseCostObservation`；
* 而当前 workflow / evidence 仍把 `pre-release` 视为单一 attested `pre_release_run_report`；因此这条 unresolved cost gate 一旦进入 current-head rerun，就会和 `TEST-OPS-001` 共用同一个 pre-release attestation 成败面，而不再只是孤立地卡住 `EVID-COST-001`；
* 因而 `TEST-COST-001` pre-release half 继续未完成，`EVID-COST-001` 当然也继续未完成。

## Why It Matters

* [41-observability-performance-and-cost.md](/root/Matrix/spec/framework/41-observability-performance-and-cost.md) 已明确要求：若任一官方 surface、时间窗口或 usage-model 前提缺失，`PreReleaseCostObservation` 必须 fail-closed，而不是用估算值补洞。
* [43-testing-and-compliance.md](/root/Matrix/spec/framework/43-testing-and-compliance.md) 与 [44-verification-and-evidence-register.md](/root/Matrix/spec/framework/44-verification-and-evidence-register.md) 都要求 `TEST-COST-001` / `EVID-COST-001` 使用 dedicated non-local suite + attested `PreReleaseCostObservation`，不是“结构合法的估算 JSON”。
* 如果当前 repo 继续把 GraphQL-derived estimates 写成 `actual_total_usd`，或让 `modeled_total_usd == actual_total_usd` 只是因为两者都来自同一套估算，就会把 fail-closed evidence gate 重新打穿。
* 当前 release-gate attestation 是按 environment 聚合的；因此只要 cost proof 仍未闭环，current-head 的 `pre-release` attestation 也不能被诚实宣称为完整通过，`EVID-DER-001` / `EVID-OPS-001` 也会继续缺少可消费的 same-head pre-release attestation。

## Options

1. Land a billing-safe, environment-scoped official proof surface
   找到并验证一条能对当前 pre-release environment + bounded workload window 直接产出 truthful cost observation 的官方 surface，然后把 `IF-OPS-010` / tooling / suite / attestation 接完。

2. Keep the dedicated suite and mapping, but keep the route fail-closed
   保留当前已经落地的 canonical suite / strict mapping / sidecar parser，继续把 `/_ops/v1/cost/observation` 维持在诚实 fail-closed 状态，直到官方 surface 真正满足 contract。

3. Narrow the current release boundary by decision
   若项目决定当前阶段不再把 `TEST-COST-001` pre-release half 保持在 `L1` 范围内，则必须通过新的 `DEC-ID` 明示收缩边界，并同步更新 `41`,`43`,`44`,`TODO.md` 与 evidence gate。

## Current Recommendation

当前建议保持本问题 open，并采用 Option 2：

* 不回退 dedicated canonical suite / strict mapping / sidecar parser；
* 不再保留任何把 GraphQL analytics estimate 冒充 `actual_total_usd` 的 success path；
* `/_ops/v1/cost/observation` 必须继续 fail-closed，直到官方 surface 真正满足 contract；
* 在 `pre_release_run_report` 仍是单一环境 attestation 的当前 topology 下，必须同时把它写清为 active gate blocker，而不是再把 cost 说成“只影响 EVID-COST-001”；
* `OQ-0002` 继续只代表 production monthly snapshot / pricing semantics blocker，不得与本问题混写，但两者都会继续阻塞 `EVID-COST-001`。

## Closure Criteria

本问题关闭前，必须至少满足以下条件之一：

* 当前仓库已证明存在 official、billing-safe、environment-scoped 的 pre-release cost proof path，并由 attested run 真实产出 `PreReleaseCostObservation`；
* `TEST-COST-001` pre-release half 已由 dedicated canonical suite + attested run 诚实转为 `pass`；
* 或项目通过新的 `DEC-ID` 明确缩减当前 release boundary，并同步更新 `41`,`43`,`44`,`TODO.md` 与 evidence gate。

## Question

`TEST-COST-001` 在 [43-testing-and-compliance.md](/root/Matrix/spec/framework/43-testing-and-compliance.md) 中被定义为 `monthly + pre-release` 门禁，语义是“quota accounting and budget guardrail validation”，要求“对比实际指标与定价模型”。`EVID-COST-001` 在 [44-verification-and-evidence-register.md](/root/Matrix/spec/framework/44-verification-and-evidence-register.md) 里也明确要求 `prod + pre-release` 两部分同时成立。

截至 `2026-04-02` 本轮，当前 Spec stack 已把 pre-release half 的 contract 收敛到 `PreReleaseCostObservation`，但实现与官方 metrics access 仍未闭合：

* `OQ-0002` 记录了 production monthly cost snapshot automation 与 Cloudflare observability pricing semantics 仍未收敛；
* evidence / attestation contract 已要求 `prod_cost_snapshot` 必须是单独的 `ProdCostSnapshotAttestation`；
* `41/43/44/26` 现已把 pre-release half 规范为：bounded workload 之后，必须对同一 environment/window 查询 official Cloudflare metrics/billing surfaces，并 attestation 化为 `PreReleaseCostObservation`；
* 但 repo 代码、workflow 与 secrets 仍未证明自己可以稳定拿到这些 official metrics surfaces。

当前 remaining gap 已从“没有 contract”收敛为“contract 已写清，但实现/权限/验证还没闭合”：

* `ops-worker` 与 workflow 还没有把 `IF-OPS-010` / `DATA-OPS-013` / `FLOW-OPS-COST-OBSERVATION` 接通；
* 当前 repository secret truth 只证明了 Access 相关权限更新，尚未证明同一个 token 或独立 token 已具备 official Cloudflare metrics/billing query access；
* `tests/pre-release/` 仍缺 dedicated canonical `test-cost-001.test.mjs`，`packages/testing/src/evidence.mjs` 也还没有 pre-release mapping。

因此，对当前 repo 来说，本项已达到 `SR2`，但实现仍受 official metrics access / validation closure 限制。

## Why It Matters

* [41-observability-performance-and-cost.md](/root/Matrix/spec/framework/41-observability-performance-and-cost.md) 的 `REQ-OPS-003` 要求“每个主要成本面都必须有可观测指标与月度归因面板”；如果 pre-release 侧没有 contract，`TEST-COST-001` 就会退化成空壳。
* `OQ-0002` 只解决 production monthly snapshot 与 Cloudflare pricing semantics 的问题，并没有定义 pre-release suite 应该消费什么“实际指标”。
* 如果不把这个 contract gap 写出来，后续很容易把 `TEST-COST-001` 继续误表述成“只差 canonical file / mapping”。

## Options

1. Keep the pre-release half of `TEST-COST-001` fail-closed until implementation and official metrics access exist
   不新增 placeholder suite，先承认 current repo 仍未把 official metrics/billing query 与 attested proof 接通。

2. Define a pre-release cost telemetry/export contract
   为 pre-release suite 新增可审计的 metrics/cost-surface export contract，并在 `26` / `IF-*` / `DATA-*` / workflow 中接通。

3. Narrow the current release boundary by decision
   通过新的 `DEC-ID` 把 pre-release cost proof 从当前 Phase 08 / `L1` scope 中移出。

## Current Recommendation

当前建议保持本问题 open，并采用 Option 2：

* contract 已经进入 Spec stack，后续应直接实现 official-metrics query / normalization / suite / attestation；
* 在 official metrics access 和 attested run 证实之前，`TEST-COST-001` / `EVID-COST-001` 仍必须 fail-closed；
* `OQ-0002` 继续只代表 production monthly snapshot 与 pricing semantics blocker，不得再和 pre-release half 混写。

## Closure Criteria

本问题关闭前，必须至少满足以下条件之一：

* `26` / `IF-*` / `DATA-*` 已定义 pre-release cost proof contract，相关实现与 dedicated suite 已落地，并由 attested run 证明；
* `TEST-COST-001` 的 pre-release half 已能诚实消费 actual metrics/cost surfaces 并通过；
* 或项目通过新的 `DEC-ID` 明确缩减当前 release boundary，并同步更新 `41`,`43`,`44`,`TODO.md` 与 evidence gate。

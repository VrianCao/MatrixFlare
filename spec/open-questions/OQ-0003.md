## Question

历史上，对 `ci-integration` / `staging` / `pre-release` 非本地运行工件与 production monthly cost snapshot，仓库只定义了“必须有非本地工件”和“人工步骤需记录执行者/审查者”的最低门槛，但还没有定义足以证明“这份 JSON 确实来自近真实 Cloudflare 拓扑执行，而不是手工伪造”的最终 attestation / trust model。

在关闭前，工具链已经 fail-closed 到以下基线：

* 同一 `run_timestamp` 不可覆写；
* 本地 `evidence/common/_test-runs/<ts>/*.json` 不得充作 non-local manual artifact；
* 仍扩展 `tests/local/*` 的薄 non-local harness 报告必须失效；
* non-local run report / prod snapshot 必须携带执行者、审查者、source URI、topology/resource 上下文与最小成本字段。

该问题现已由 [DEC-0002](/root/Matrix/spec/decisions/DEC-0002.md) 收敛：release-gate non-local evidence 必须通过 attestation bundle 导入，并强制携带 run identity、deployment identity、audited artifact storage immutable reference 与 review record reference；自由形状 run report / prod snapshot 不再是可信入口。当前仓库仍未完成 `08.04` / `08.05`，但未完成原因也已从“provenance model 未定义”收敛为两层真实约束：一是 run `23869349481` 证明了三环境 deploy/upload/attestation 基础链路可工作；二是更新的 run `23879497392` 又证明当前分支代码状态仍可能在 dedicated non-local suite 上真实失败，其中 `staging` 与 `pre-release` 都在 `TEST-MEDIA-001` 中把媒体下载正文读成了 `'[object ReadableStream]'`，因此不存在“当前代码状态已有三环境 attested pass”的诚实说法。真正剩余的是完整 non-local 语义覆盖、当前代码状态的最新 attested pass，以及 production cost evidence，而不是 provenance model 未定义。

## Why It Matters

* [43-testing-and-compliance.md](/root/Matrix/spec/framework/43-testing-and-compliance.md) 第 5 节要求在接近 Cloudflare 真实拓扑验证，且 mock 结论不得作为发布门禁。
* [44-verification-and-evidence-register.md](/root/Matrix/spec/framework/44-verification-and-evidence-register.md) 要求 `pass` 证据可审计，并对人工步骤记录执行者/审查者。
* `TODO.md` 的 `08.04` / `08.05` 当前明确保持未完成，直到真实 attested non-local runs、完整 non-local 语义覆盖与 production monthly cost evidence 存在；若 provenance 模型不清，未来很容易再次出现“工件存在但证据不诚实”的回退。

## Options

1. Trust manual provenance fields only
   继续只依赖 `executed_by` / `reviewed_by` / `source_run_uri` / topology metadata，接受人工审核作为最终信任根。

2. Require pipeline-emitted attestation bundles
   由真实 CI/staging/pre-release harness 生成带签名或可验证 digest 的 attestation bundle，并把 run report 视为该 attestation 的派生视图。

3. Require deploy/run identity plus audited artifact storage
   要求工件必须回链到 Cloudflare deployment/run identifiers、固定 artifact store 对象键、以及受审计的 review records；即使不做签名，也必须让 provenance 具备外部可核对的不可变引用。

## Current Recommendation

本问题的历史建议已由 [DEC-0002](/root/Matrix/spec/decisions/DEC-0002.md) 取代。当前唯一有效结论是：

* non-local evidence 只能通过 `EnvironmentRunAttestation` / `ProdCostSnapshotAttestation` 进入 evidence gate；
* attestation 必须绑定 origin run identity、deployment identity、artifact store immutable reference 与 audited review record；
* 即使 dedicated non-local harness 已落地，在真实 attested runs、完整 non-local 语义覆盖与 production cost evidence 闭合前，`08.04` / `08.05` 仍必须保持未完成。

## Closure Criteria

本问题已关闭，闭环条件如下：

* [DEC-0002](/root/Matrix/spec/decisions/DEC-0002.md) 已固定当前最小可信 provenance model；
* `26/43/44` 与 evidence toolchain 已同步改为 attestation-bundle contract；
* `TODO.md` 已改回真实状态，不再把自由形状 non-local JSON 当作未来可直接接受的发布证据。

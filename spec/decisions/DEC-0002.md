## Decision

所有用于 release gate 的 non-local 运行工件与 production monthly cost snapshot，必须通过权威 attestation contract 导入 evidence gate；不得再把自由形状 JSON run report / cost snapshot 直接当作发布证据。

现行真相固定如下：

* 非本地测试环境 `ci-integration`、`staging`、`pre-release` 必须分别产出 [26-wire-schema-catalog.md](/root/Matrix/spec/framework/26-wire-schema-catalog.md) 中定义的 `EnvironmentRunAttestation`。
* production monthly cost snapshot 必须产出同一分册定义的 `ProdCostSnapshotAttestation`。
* attestation bundle 必须同时包含：
  * canonical payload（即 environment run report 或 prod cost snapshot）；
  * origin run identity；
  * deployment identity；
  * audited artifact storage immutable reference；
  * review record reference。
* 对当前 Phase 08 acceptance model，consumer 还必须 fail-closed 于以下 provenance 失配：
  * `origin_system` 不是 `github-actions`；
  * `origin_repository` 为空、不是 GitHub `owner/repo` slug，或与 `origin_run_uri` 中编码的 repository 不一致；
  * consumer 已能解析当前仓库 identity，但 imported attestation 的 `origin_repository` 与当前仓库不一致；
  * `origin_run_uri` 不是 GitHub Actions run URL；
  * `artifact_store_uri` 不是 `r2://bucket/key` immutable object locator；
  * `artifact_store_key` 与 `artifact_store_uri` 不一致；
  * `deployment_identity.environment_id` 不等于 attestation `source_environment`。
* 上述 provenance / payload 中所有 `_uri` locator 都必须是可解析的绝对外部 URI / locator，且需具 authority，或使用格式完整的 `urn:<nid>:<nss>`；裸 `urn:`、占位字符串、`about:`、`blob:`、`file:`、`data:`、`javascript:` 等本地或不可审计引用不得视为可信 provenance。
* evidence gate 必须 fail-closed 于以下任一情况：
  * 只提供原始 run report / prod snapshot，而不是 attestation bundle；
  * attestation 缺少 run/deployment identity、artifact store reference、review record、digest 或 schema version；
  * attested payload 与目标 `run_timestamp`、目标环境或 canonical payload schema 不一致；
  * non-local harness 仍扩展 `tests/local/*`、存在 repo boundary escape，或存在 unresolved dynamic import；
  * 导入路径来自 `evidence/common/_test-runs` 或其 alias/hard-link。

本决策采用 [OQ-0003](/root/Matrix/spec/open-questions/OQ-0003.md) 的 Option 3，即 “deploy/run identity + audited artifact storage” 作为当前最小可审计 provenance model；当前不要求额外签名体系，但也不得把缺少上述 provenance 链条的 JSON 视为可信 non-local evidence。

## Rationale

* 仅靠 `executed_by` / `reviewed_by` / `source_run_uri` / topology metadata，仍无法把“结构合法但来源不可信”的 JSON 与真实 non-local 执行区分开。
* 在本决策被接受时，仓库还没有可复用的 dedicated Cloudflare-topology harness；因此必须先把未来可接受的 provenance contract 钉死，避免再次把薄入口或本地产物误算为 Phase 08 完成依据。之后仓库虽已接入 dedicated non-local harness 与 GitHub Actions workflow，并在 `2026-04-01` 的最新验证 run `23869349481` 拿到 `ci-integration` / `staging` / `pre-release` 三环境成功 attested runs，但完整 non-local L1 语义覆盖与 production cost evidence 仍未闭环。
* 与要求即时引入签名服务或在线外部存储回读相比，run identity + deployment identity + artifact store immutable reference + review record 更符合当前项目阶段，也足以让 evidence gate 在结构层面 fail-closed；更强的在线 provenance dereference / digest re-check 不属于本决策当前要求。

## Consequences

* [OQ-0003](/root/Matrix/spec/open-questions/OQ-0003.md) 关闭；非本地 evidence provenance 不再是开放式语义。
* `26/43/44` 与 evidence toolchain 必须把 attestation contract 当作唯一可接受的 non-local artifact 入口。
* 这不会自动完成 `08.04` / `08.05`，因为即使 dedicated non-local harness 与 workflow 已接通、且 `2026-04-01` 的最新验证 run `23869349481` 已产出三环境成功 attested runs，完整 non-local L1 语义覆盖与 production cost evidence 仍然缺失；它只收敛了“将来什么才算可信工件”的规范边界。

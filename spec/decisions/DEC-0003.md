## Decision

当前项目正式接受 account-owned `workers.dev` 上的 Cloudflare Access path 作为 `staging` / `pre-release` 非本地 `/_ops` management ingress 的有效 closure basis；不再把 active-zone custom hostname 视为 `08.04B` 的唯一合法实现路径。

该接纳以 current-head GitHub Actions attested run `23923285726`（head `6fc23f493b5a19cfc2ca20993cc0772ee06a8754`）为审计基线，要求同时满足以下事实：

* `staging` / `pre-release` 的 `Prepare non-local ops Access session` 成功；
* Access-protected `/_ops` readiness probe 成功；
* `/_ops/v1/rebuilds` 在真实 Cloudflare managed D1 上不再被 raw transaction SQL 卡住；
* `staging` / `pre-release` 的 `EnvironmentRunAttestation` 均有效；
* 同轮 `EVID-DER-001` 转为 `pass`。

本决策同时固定以下边界：

* `08.04B` / `OQ-0004` 的 closure 只证明 non-local ops Access topology、automation contract 与 Access-authenticated rebuild path 已经真实闭环；
* 该 closure 不能被误写成 `TEST-OPS-001` / `EVID-OPS-001` 已具备 rollout-skew proof；
* `TEST-OPS-001` 的 rollout-skew raw same-head proof 已在后续 [OQ-0005](/root/Matrix/spec/open-questions/OQ-0005.md) 中被关闭；`EVID-OPS-001` 当前仍为 `fail` 的原因已转为 shared pre-release attestation 继续被 [OQ-0006](/root/Matrix/spec/open-questions/OQ-0006.md) 的 cost gate 阻断。

## Rationale

* Cloudflare 官方 Access 与 Workers 文档当前同时允许在 account-owned `workers.dev` / Preview URL 上启用 Cloudflare Access，因此 active-zone custom hostname 不是唯一合法路径。
* 仓库已经把 workers.dev Access automation contract 落到了 `packages/testing/src/nonlocal.mjs` 与 `.github/workflows/nonlocal-phase08.yml`，包括 org lookup、application ensure、service-token rotate/create、service-auth policy ensure、远端 `DATA-D1-006` bootstrap 与 workflow session handoff。
* 在 run `23918727505` 中，真正暴露的 blocker 已经不是 Access topology，而是 managed-D1 rebuild compatibility；当前 head 的 attested run `23923285726` 则进一步证明该 repo-side blocker 已被收口。
* 若没有本决策，仓库会继续把已经被 attested run 证明成立的 workers.dev path 当作 open question，导致 `TODO.md`、`15-source-observation-register.md` 与 `OQ-0004` 继续陈述过时真相。

## Consequences

* [OQ-0004](/root/Matrix/spec/open-questions/OQ-0004.md) 关闭；
* `TODO.md` 的 `08.04B` 可以诚实标记为完成；
* `EVID-DER-001` 的 current-head non-local proof 现在被视为有效 closure，而不再只是待确认候选；
* `08.04` 与 `08.05` 仍未完成，因为 cost-related blocker 仍然存在；`EVID-OPS-001` 也会继续因 shared pre-release attestation 缺失而 fail-closed；
* 若后续 workers.dev Access 文档语义、平台行为或 attested run 结果发生漂移，必须通过新的 `OQ-ID` 或替代 `DEC-ID` 重新审查，而不是静默回退到 placeholder 叙述。
